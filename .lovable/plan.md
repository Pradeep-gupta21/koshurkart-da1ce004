
# Vercel Deployment Readiness Plan

Goal: make this Vite + React + Supabase project deploy cleanly on Vercel without changing design or functionality.

## What I'll add / change

1. **`vercel.json`** (new) — configure SPA rewrites and build settings so deep links (e.g. `/vendor/apply`, `/admin/...`) don't 404 on refresh:
   ```json
   {
     "buildCommand": "npm run build",
     "outputDirectory": "dist",
     "framework": "vite",
     "rewrites": [{ "source": "/(.*)", "destination": "/index.html" }]
   }
   ```
   Required because the app uses `BrowserRouter` with many nested routes (`/admin/*`, `/vendor/*`, `/account/*`, payments, etc.).

2. **`.env.example`** (new) — document the three required public env vars so the user knows what to paste into Vercel → Project → Settings → Environment Variables:
   - `VITE_SUPABASE_URL`
   - `VITE_SUPABASE_PUBLISHABLE_KEY`
   - `VITE_SUPABASE_PROJECT_ID`

   (These are publishable keys, safe to expose. The Lovable Cloud values from `.env` will be reused.)

3. **Production build verification** — run `npm run build` once to confirm:
   - TypeScript compiles
   - Vite produces a `dist/` output
   - No missing imports or broken paths
   
   If anything fails, fix only what's needed to make the build succeed (no design/behavior changes).

4. **README deployment note** (small append) — short "Deploy on Vercel" section listing the env vars and that the SPA rewrite is handled by `vercel.json`.

## What I will NOT change

- `vite.config.ts` — already Vercel-compatible (standard Vite output to `dist/`).
- `src/integrations/supabase/client.ts` — auto-generated, untouched.
- `.env` — auto-generated by Lovable Cloud, untouched.
- Any UI, routes, components, edge functions, or business logic.
- Supabase configuration (already correctly set up via Lovable Cloud; the same backend will serve the Vercel-hosted frontend).

## Deployment readiness report (delivered after build verification)

I'll report:
- Build status (pass/fail + bundle size summary)
- Env vars the user must set in Vercel
- Confirmation SPA routing is handled
- Any blockers found and how they were fixed
- Step-by-step Vercel deploy instructions (import repo → set env vars → deploy)

## Notes

- Edge functions stay on Lovable Cloud (Supabase). Vercel only hosts the static frontend. No serverless functions need to migrate.
- Custom domain for auth emails (`notify.notify.koshurkart.shop`) is independent of where the frontend is hosted — it keeps working.
- Razorpay, UPI, and other integrations call Lovable Cloud edge functions directly from the browser, so they work identically from a Vercel-hosted frontend.
